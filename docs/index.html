The problem

Large codebases don't fit in a context window.

Reading whole files to answer "what depends on this?" burns tokens and misses cross-file links. AST-MCP gives the agent structure first — signatures, imports, and a real dependency graph — so it reasons precisely and reads only what matters.

Quick start

Up and running in three steps.

1

Install & build

Clone, install deps — dist/ builds automatically.

2

Try the CLI

Run analysis on any source directory.

3

Wire up the MCP server

Add it to Claude Desktop and let the agent analyze.

terminal
# clone & install — postinstall builds dist/ for you
git clone https://github.com/6ixthxense/AST-MCP.git
cd AST-MCP
npm install

# sanity check
ast-map --help
ast-map langs
terminal
# find unused exports
ast-map dead src/

# circular imports?
ast-map cycles src/

# blast radius of changing sanitize()
ast-map impact src/utils.ts sanitize --scan src/

# don't know the name? search by meaning
ast-map find "remove expired cache entries" src/

# one-page health dashboard (A–F grade)
ast-map report src/ -o health.html
claude_desktop_config.json
{
  "mcpServers": {
    "ast-mapper": {
      "command": "node",
      "args": ["C:\\path\\to\\AST-MCP\\dist\\index.js"],
      "env": {
        "AST_MAP_ROOT": "C:\\path\\to\\your\\project"
      }
    }
  }
}
// AST_MAP_ROOT is the security boundary — the
// server only reads files inside this path.
Tool catalog

A focused tool for every question.

A sample of the 44 MCP tools. Each returns clean JSON an agent can act on directly.

get_symbol_context

Exact source lines of a named symbol — a 300-line file down to ~40. Pull related types too.

build_symbol_graph

Two-layer graph: file + symbol nodes, imports + contains edges. The backbone of every analysis.

get_change_impact

Reverse-traverse imports to compute the blast radius of a symbol — direct + transitive dependents.

semantic_search

Find symbols by meaning, not name. Tokenization + programming thesaurus + BM25 ranking. No embeddings.

scan_security

12 static rules — eval, innerHTML, SQL injection, weak crypto, hardcoded secrets, prototype pollution…

get_codebase_report

One-shot health summary: A–F grade, complexity hotspots, God Nodes, dead code, cycles, coupling, SDP.

pack_context

Minimal context to change a symbol: its source + dependency signatures + dependents, with a token estimate.

get_test_coverage

Pair tests with the sources they exercise and rank untested files by risk — no test runner needed.
